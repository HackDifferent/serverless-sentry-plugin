# Commit Conventions

## Format

No strict commit format is enforced. The repository uses plain English descriptions. No ticket ID prefix, no conventional commits type prefix as a requirement.

**Pattern**: `[Optional type prefix: ] Brief description`

## Examples from git log

```
Fixed a typo in the README.md
Increased number of parallel uploads of source map artifacts for better performance.
Updated changelog
Made comment on catching errors in the exactTag() function more explicit
Prepare 2.5.1
Source Map fixes
Cleaned up README.md
Added configuration validation
fix: validation-schema          ← occasional conventional commits style from contributors
Add support for uploading sourcemap files
use the enabled option
```

## Version Bump Commits

Version bump commits are just the version number:
```
2.5.3
2.5.2
2.5.1
2.5.0
2.4.0
2.3.0
2.2.0
2.1.0
2.0.2
2.0.1
2.0.0
```

These are generated by `npm version` and should not be hand-crafted.

## PR Merge Commits

External PRs use GitHub's default merge commit format:
```
Merge pull request #53 from syatt-io/add-source-map-uploads
Merge pull request #56 from DonaldoLog/fix/validation-schema
```

## Guidance

- Write in imperative mood preferred but not required ("Fix bug" not "Fixed bug" — both appear in history)
- No ticket IDs
- No required body or footer
- Keep messages concise and descriptive
